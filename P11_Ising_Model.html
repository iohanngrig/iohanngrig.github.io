Ising Model - Enumerations and Monte Carlo Algorithms¶

${\bf \large \cal Hovhannes \ Grigoryan}\\ {\rm \small New \ York, \ NY}$

Abstract

We start with Gray algorithm that lists all possible spin configurations, and use these to calculate density of states, and thermodynamic quantities such as mean energy and specific heat capacity per particle. To be able to extend calculations to large sizes, we consider local Monte Carlo sampling algorithms. We use Metropolis Monte-Carlo method to study phase transition between paramagnet and ferromagnet. Next, we discuss heat bath algorithm, and demonstrate the phenomenon of coupling. Finally, we consider cluster algorithm, which solves the critical slowing down problem near the critical temperature present in Metropolis Monte Carlo method [1].

Content

  • Ising Model - exact computations
  • Thermodynamics, specific heat capacity, and magnetization
  • Local Monte Carlo sampling
  • Heat bath algorithm
  • Coupling
  • Cluster Algorithm

Ising Model - exact computations¶

The Ising model describes spins $\sigma_k = \pm 1$, for $k = 1, \dots, N$, on a lattice. In the ferromagnetic Ising model, neighboring spins prefer to align, since the energy is:

$$E = -J \sum_{\langle k,m \rangle} \sigma_k \sigma_m ,$$

where $J>0$, and the sum is over all distinct pairs of neighbors. For simplicity we will set $J=1$. This simple system shows order disorder transition in $D \geq 2+1$, and can be solved analytically for $D=2+1$. The Ising model can also describe particles on a lattice, with $\sigma_k= 1, 0$ signaling the presence or absence of a particle on site k.

Let's define a function Nbr$(k, n)$, which finds the $n$-th neighbor of a spin at position $k$. In 2D Ising model, neighbor scheme is as follows:

$$ \text{site enumeration} = \begin{vmatrix} 7 & 8 & 9 \\ 4 & 5 & 6 \\ 1 & 2 & 3 \\ \end{vmatrix} $$$$ \text{neighbor enumeration} = \begin{vmatrix} & 2 & \\ 3 & x & 1 \\ & 4 & \\ \end{vmatrix} $$

Therefore, for instance, Nbr$(2,1) = 3$, the first neighbor of site 2 is to the right of it, and is site 3. Similarly, the forth neighbors (below) for sites 2 and 5 are: Nbr$(2,4) = 0$, while Nbr$(5,4) = 2$.

def energy_ising(S, N, nbr, J=1.0):
    """ S - spin list/array (flatten to 1d, even if d>1), 
        N - total number of spins on a lattice,
        nbr - neighbor scheme dict or list of lists/tuples, e.g. in 2d, 
              nbr[n-th spin position] = (right, up, left, down) neighbor positions, 
        J - spin-spin interaction strength."""
    E = 0
    for n in range(N):
        E -= J * S[n] * sum([S[j] for j in nbr[n]])
    return 0.5 * E

Listing spin configurations¶

All $2^N$ spin configurations can be enumerated through a sequence of $2^N$ spin-flips, one at a time. Algorithms that perform such enumerations are called Gray codes.

In [6]:
@jit
def gray_flip(t, N):
    """t - is a list of type [1, ..., N+1], chosen initially.
       k - next spin to flip."""
    k = t[0]
    if k > N: 
        return t, k
    t[k - 1] = t[k]
    t[k] = k + 1
    if k != 1: 
        t[0] = 1
    return t, k

Density of states¶

The advantage of calculating the density of states, $\mathcal{N}(E)$, is that we can calculate it once, and use it to calculate any thermodynamic quantity later.

In [10]:
def density_of_states(L):
    """Density of states for periodic Ising model of size L x L."""
    N = L * L
    nbr = {i: ((i // L) * L + (i + 1) % L, (i + L) % N,
               (i // L) * L + (i - 1) % L, (i - L) % N)
           for i in range(N)}
    S = [-1] * N
    E = -2 * N
    dos = {}       # density of states N(E)
    dos[E] = 1

    tau = list(range(1, N + 2))

    for i in range(1, 2 ** N):
        tau, k = gray_flip(tau, N)
        h = sum([S[n] for n in nbr[k - 1]])  # field on site k-1
        E += 2 * h * S[k - 1]
        S[k - 1] *= -1
        if E in dos:
            dos[E] += 1
        else:
            dos[E] = 1
    return dos

Thermodynamics, specific heat capacity, and magnetization¶

The partition function for the Ising-model, expressed in terms of density of states can be written as:

$$Z(\beta) = \sum_{\sigma \in \{\sigma_j\}_{j=1}^N} e^{-\beta E(\sigma)} = \sum_E \mathcal{N}(E) e^{-\beta E}$$

The mean energy, and mean energy square are:

$$\langle E^j \rangle = \frac{1}{Z} \sum_E E^j \mathcal{N}(E) e^{-\beta E}, \ \ \ \ j={1,2}$$

Therefore, the specific heat capacity per particle is:

$$c_V = \frac{1}{N} \frac{\partial \langle E \rangle}{\partial T} = -\frac{\beta^2}{N}\frac{\partial\langle E \rangle}{\partial \beta} = \frac{\beta^2}{N} \left(\langle E^2 \rangle - \langle E \rangle^2 \right)$$

def thermo_ising(dos, beta):
    Z = 0.0
    E_av = 0.0
    E2_av = 0.0
    for E in dos.keys():
        weight = np.exp(- beta * E) * dos[E]
        Z += weight
        E_av += weight * E
        E2_av += weight * E ** 2

    E_av /= Z
    E2_av /= Z
    cv = (beta ** 2) * (E2_av - E_av ** 2) / float(N)
    E_av = E_av / float(N)
    return Z, E_av, cv


def specific_heat(dos, list_T):
    list_cv = []
    for T in list_T:
        Z, E_av, cv = thermo_ising(dos, 1.0/T)
        list_cv.append(cv)
    return list_cv

Local Monte Carlo Sampling¶

Often, Monte Carlo methods alone are able to obtain exact results for large sizes. The price to be paid is that configurations are sampled rather than enumerated, so that statistical errors are inevitable.

Metropolis Monte-Carlo method¶

Consider spin configuration $a$. Pick a random spin and flip it. Let's call this new configuration $b$. As usual, we accept $b$ with Metropolis acceptance probability:

$$p^{\text{acc}}(a \to b) = \min\left(1, e^{-\beta(E_b - E_a)} \right)$$

This method allows to observe a phase transition between ferromagnet and paramagnet at $T_c \approx 2$. However, near the critical temperature, because of large spin-spin correlation length, flipping spin becomes increasingly difficult, and algorithms slows down significantly. This is known as critical slowing down.

Heat bath algorithm¶

Metropolis-Hastings algorithm: The general form of the detailed balance condition can be written as:

$$\pi(a)\mathcal{A}(a \to b) p^{\text{acc}}(a \to b) = \pi(b)\mathcal{A}(b \to a) p^{\text{acc}}(b \to a)$$

In Metropolis algorithm, a priori probabilities are symmetric: $\mathcal{A}(a \to b) = \mathcal{A}(b \to a)$, and the detailed balance condition reduces to:

$$\pi(a) p^{\text{acc}}(a \to b) = \pi(b) p^{\text{acc}}(b \to a)$$

However, in the heat bath algorithm, $\mathcal{A}(a \to b) \propto \pi(b)$ and $\mathcal{A}(b \to a) \propto \pi(a)$. Therefore, $p^{\text{acc}}(a \to b) = p^{\text{acc}}(b \to a) = 1$.

In the heat bath algorithm, rather than flipping a spin at a random site, we now thermalize this spin with its local environment. In the presence of a molecular field $h$ at site $k$, the spin points up and down with probabilities:

$$\pi^{\pm}_h = \frac{e^{-\beta E^{\pm}}}{e^{-\beta E^{+}} + e^{-\beta E^{-}}} = \frac{1}{1 + e^{\mp 2\beta h}}$$

To sample $\{\pi^{-}_h , \pi^{+}_h \}$, pick a random number $\Upsilon = \text{ran}(0,1)$, and point spin up if $\Upsilon < \pi^+_h$, and down otherwise.

def heatbath_ising(L, beta, nsteps, graph=False):
    N = L * L
    nbr = {i: ((i // L) * L + (i + 1) % L, (i + L) % N,
           (i // L) * L + (i - 1) % L, (i - L) % N) for i in range(N)}
    
    # initial configuration (if final configs are independent of initial config, we have coupling)
    S = [random.choice([-1, 1]) for i in range(N)]
    E = -0.5 * sum([S[k] * sum([S[nn] for nn in nbr[k]]) for k in range(N)])
    E_tot, E2_tot = 0.0, 0.0

    random.seed("123456")  # to make sure that algo depends on sites but not on initial config
    for step in range(nsteps):
        k = random.randint(0, N - 1)
        Upsilon = random.uniform(0.0, 1.0)
        h = sum(S[nn] for nn in nbr[k])
        Sk_old = S[k]
        S[k] = -1

        pi_plus = 1.0 / (1.0 + np.exp(-2.0 * beta * h))
        if Upsilon < pi_plus: 
            S[k] = 1
        if S[k] != Sk_old:
            E -= 2.0 * h * S[k]
            
        E_tot += E
        E2_tot += E ** 2
                
        if graph:
            if step % 10000 == 1:
                clear_output(True)
                S_conf = np.array(S).reshape(L, L)

Coupling¶

The essence of coupling is that different initial configurations lead to the same final configuration. This verifies that the system reached the equilibrium. Markov chains couple, and forget their initial configurations. This happens in a timescale of order of the correlation time. Therefore, the coupling time must be larger that the correlation time.

Cluster Algorithm (Wolff '89)¶

Near the critical temperature, cluster algorithms speed up the calculations significantly. In this method the spin chain of the size of the system is being flipped. We select a site, and construct cluster of connected spins, then we flip all spins in this cluster. We accept a spin into the cluster with some probability, $p$. The algorithm should obey the detailed balance condition:

$$\pi(a) P(a \to b) = \pi(b) P(b \to a)$$$$P(a \to b) = \mathcal{A}(a \to b) p^{\text{acc}}(a \to b)$$

$$P(b \to a) = \mathcal{A}(b \to a) p^{\text{acc}}(b \to a)$$

where $\pi(a)$ is a statistical weight of configuration $a$, $P(a \to b)$ is the probability to move from $a$ to $b$, $\mathcal{A}(a \to b)$ is the probability to construct cluster $b$ from $a$, and $p^{\text{acc}}(a \to b)$ is the acceptance probability.

The a priori probability $\mathcal{A}(a \to b) \propto (1-p)^{\text{number of ++ edges}}$. Similarly, $\mathcal{A}(b \to a) \propto (1-p)^{\text{number of -- edges}}$. The energy across the boundary of cluster in $a$ is $n_1-n_2$, where $n_1$ is the number of $+-$ links, and $n_2$ is the number of $++$ links. Similarly, the energy across the boundary in cluster $b$ is $n_2-n_1$. Therefore, the detailed balance condition can be rewritten as:

$$e^{-\beta(n_1 - n_2)} (1-p)^{n_2}p^{\text{acc}}(a \to b) = e^{-\beta(n_2 - n_1)} (1-p)^{n_1}p^{\text{acc}}(b \to a)$$

As a result, the Metropolis acceptance probability should be:

$$p^{\text{acc}}(a \to b) = \min\left[1, \left(\frac{e^{-2\beta}}{1-p}\right)^{n_2}\left(\frac{e^{-2\beta}}{1-p}\right)^{-n_1} \right]$$

Therefore, for $1 - p = e^{-2\beta}$ we have no rejections, so we can construct a cluster, and flip it, construct another cluster accepting a spin to a cluster with probability $p$, and flip it again, etc. This is the essence of Wolff cluster algorithm.

def cluster_algo(L, T, nsteps):
    N = L * L
    nbr = {i: ((i // L) * L + (i + 1) % L, (i + L) % N,
               (i // L) * L + (i - 1) % L, (i - L) % N)
           for i in range(N)}

    p = 1.0 - np.exp(-2.0 / T)

    S = [random.choice([1, -1]) for k in range(N)]
    plt.rcParams['figure.figsize'] = (12,12) 
    
    for step in range(nsteps):
        # pick a random spin and put it in a cluster and into a pocket
        k = random.randint(0, N - 1)
        Pocket, Cluster = [k], [k]

        while Pocket != []:
            j = random.choice(Pocket)
            # take a random element from pocket, and checking all of its neighbors
            for l in nbr[j]:
                # for each neighbor check: a) if it has same sign; b) not already in a cluster; c) p > ran(0,1)
                if (S[l] == S[j]) and (l not in Cluster) and (random.uniform(0.0, 1.0) < p):
                    Pocket.append(l)
                    Cluster.append(l)

            # once checked the neighbors of spin j, take it out of the pocket
            Pocket.remove(j)

        # once the pocket is empty, flip the entire cluster
        for j in Cluster:
            S[j] *= -1

        if step % 10 == 1:
            clear_output(True)
            S_conf = np.array(S).reshape(L, L)
            histo = plt.imshow(S_conf, extent=[0, L, 0, L], interpolation='nearest', vmin=0, vmax=1.00)
            histo.set_cmap('hot')
            plt.grid(False)
            plt.title(f"Wolff cluster algorithm: T={T}, |M|/N={np.abs(sum(S))/float(N)}, step={step}", fontsize=22)
            filename = f"spin_cluster_t={step}.png"
            plt.savefig('../data/'+ filename, transparent=True)
            plt.show()
